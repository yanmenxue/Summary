## 1. Reduce-Scatter（规约散射）

### 目标
将所有节点的数据按位置规约（求和），然后将结果**切分**，每个节点只保留**一个位置**的完整汇总结果。

### 初始状态
每个节点持有完整的 4 块数据：

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | A0 | A1 | A2 | A3 |
| **节点 1** | B0 | B1 | B2 | B3 |
| **节点 2** | C0 | C1 | C2 | C3 |
| **节点 3** | D0 | D1 | D2 | D3 |

### 发送规则
每轮每个节点发送的块编号**向后移动一位**：

- **第 1 轮**：节点 i 发送 **位置 i**
- **第 2 轮**：节点 i 发送 **位置 (i-1) mod N**
- **第 3 轮**：节点 i 发送 **位置 (i-2) mod N**

---

### 第 1 轮：错位发送，累加接收

| 发送方 → 接收方 | 发送内容 | 接收方累加后 |
| :--- | :--- | :--- |
| 节点 0 → 节点 1 | 位置 0：A0 | 节点1 的 位置 0 变成 B0 + A0 |
| 节点 1 → 节点 2 | 位置 1：B1 | 节点2 的 位置 1 变成 C1 + B1 |
| 节点 2 → 节点 3 | 位置 2：C2 | 节点3 的 位置 2 变成 D2 + C2 |
| 节点 3 → 节点 0 | 位置 3：D3 | 节点0 的 位置 3 变成 A3 + D3 |

**第 1 轮结束后：**

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | A0 | A1 | A2 | A3 + D3 |
| **节点 1** | B0 + A0 | B1 | B2 | B3 |
| **节点 2** | C0 | C1 + B1 | C2 | C3 |
| **节点 3** | D0 | D1 | D2 + C2 | D3 |

---

### 第 2 轮：继续错位，继续累加

发送位置编号 = (i-1) mod 4，即：节点0→位置3，节点1→位置0，节点2→位置1，节点3→位置2。

| 发送方 → 接收方 | 发送内容（当前累加值） | 接收方累加后 |
| :--- | :--- | :--- |
| 节点 0 → 节点 1 | 位置 3：A3 + D3 | 节点1 的 位置 3 变成 B3 + A3 + D3 |
| 节点 1 → 节点 2 | 位置 0：B0 + A0 | 节点2 的 位置 0 变成 C0 + B0 + A0 |
| 节点 2 → 节点 3 | 位置 1：C1 + B1 | 节点3 的 位置 1 变成 D1 + C1 + B1 |
| 节点 3 → 节点 0 | 位置 2：D2 + C2 | 节点0 的 位置 2 变成 A2 + D2 + C2 |

**第 2 轮结束后：**

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | A0 | A1 | A2 + D2 + C2 | A3 + D3 |
| **节点 1** | B0 + A0 | B1 | B2 | B3 + A3 + D3 |
| **节点 2** | C0 + B0 + A0 | C1 + B1 | C2 | C3 |
| **节点 3** | D0 | D1 + C1 + B1 | D2 + C2 | D3 |

---

### 第 3 轮：最终汇聚

发送位置编号 = (i-2) mod 4，即：节点0→位置2，节点1→位置3，节点2→位置0，节点3→位置1。

| 发送方 → 接收方 | 发送内容（当前累加值） | 接收方累加后 |
| :--- | :--- | :--- |
| 节点 0 → 节点 1 | 位置 2：A2 + D2 + C2 | 节点1 的 位置 2 = B2 + A2 + D2 + C2 = **Sum2** ✅ |
| 节点 1 → 节点 2 | 位置 3：B3 + A3 + D3 | 节点2 的 位置 3 = C3 + B3 + A3 + D3 = **Sum3** ✅ |
| 节点 2 → 节点 3 | 位置 0：C0 + B0 + A0 | 节点3 的 位置 0 = D0 + C0 + B0 + A0 = **Sum0** ✅ |
| 节点 3 → 节点 0 | 位置 1：D1 + C1 + B1 | 节点0 的 位置 1 = A1 + D1 + C1 + B1 = **Sum1** ✅ |

---

### 🎯 Reduce-Scatter 最终状态

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | A0 | **Sum1** ✅ | A2 + D2 + C2 | A3 + D3 |
| **节点 1** | B0 + A0 | B1 | **Sum2** ✅ | B3 + A3 + D3 |
| **节点 2** | C0 + B0 + A0 | C1 + B1 | C2 | **Sum3** ✅ |
| **节点 3** | **Sum0** ✅ | D1 + C1 + B1 | D2 + C2 | D3 |

**关键结果：**
- 节点 0 持有 **Sum1**
- 节点 1 持有 **Sum2**
- 节点 2 持有 **Sum3**
- 节点 3 持有 **Sum0**

## 2. AllGather（全聚合）

### 目标
将 Reduce-Scatter 后分散在各节点的汇总结果，**共享给所有节点**，让每个节点都拥有完整的 `[Sum0, Sum1, Sum2, Sum3]`。

### 初始状态（Reduce-Scatter 完成后）

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | ❌ 无效 | ✅ **Sum1** | ❌ 无效 | ❌ 无效 |
| **节点 1** | ❌ 无效 | ❌ 无效 | ✅ **Sum2** | ❌ 无效 |
| **节点 2** | ❌ 无效 | ❌ 无效 | ❌ 无效 | ✅ **Sum3** |
| **节点 3** | ✅ **Sum0** | ❌ 无效 | ❌ 无效 | ❌ 无效 |

### 发送规则
每轮每个节点把**手中所有有效数据**打包发送给下一个节点，接收方直接**拼接**（覆盖无效数据）。

---

### 第 1 轮：交换手中的 1 块数据

| 发送方 → 接收方 | 发送内容 | 接收方拼接后 |
| :--- | :--- | :--- |
| 节点 0 → 节点 1 | Sum1 | 节点1 获得位置1，现持有：Sum1, Sum2 |
| 节点 1 → 节点 2 | Sum2 | 节点2 获得位置2，现持有：Sum2, Sum3 |
| 节点 2 → 节点 3 | Sum3 | 节点3 获得位置3，现持有：Sum0, Sum3 |
| 节点 3 → 节点 0 | Sum0 | 节点0 获得位置0，现持有：Sum0, Sum1 |

**第 1 轮结束后：**

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | ✅ Sum0 | ✅ Sum1 | ❌ 无效 | ❌ 无效 |
| **节点 1** | ❌ 无效 | ✅ Sum1 | ✅ Sum2 | ❌ 无效 |
| **节点 2** | ❌ 无效 | ❌ 无效 | ✅ Sum2 | ✅ Sum3 |
| **节点 3** | ✅ Sum0 | ❌ 无效 | ❌ 无效 | ✅ Sum3 |

---

### 第 2 轮：打包发送 2 块数据

每个节点把自己手里**所有有效数据**打包发给下一个节点。

| 发送方 → 接收方 | 发送内容（打包） | 接收方拼接后 |
| :--- | :--- | :--- |
| 节点 0 → 节点 1 | Sum0, Sum1 | 节点1 获得位置0，现持有：Sum0, Sum1, Sum2 |
| 节点 1 → 节点 2 | Sum1, Sum2 | 节点2 获得位置1，现持有：Sum1, Sum2, Sum3 |
| 节点 2 → 节点 3 | Sum2, Sum3 | 节点3 获得位置2，现持有：Sum0, Sum2, Sum3 |
| 节点 3 → 节点 0 | Sum0, Sum3 | 节点0 获得位置3，现持有：Sum0, Sum1, Sum3 |

**第 2 轮结束后：**

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | ✅ Sum0 | ✅ Sum1 | ❌ 无效 | ✅ Sum3 |
| **节点 1** | ✅ Sum0 | ✅ Sum1 | ✅ Sum2 | ❌ 无效 |
| **节点 2** | ❌ 无效 | ✅ Sum1 | ✅ Sum2 | ✅ Sum3 |
| **节点 3** | ✅ Sum0 | ❌ 无效 | ✅ Sum2 | ✅ Sum3 |

---

### 第 3 轮：最终补齐

| 发送方 → 接收方 | 发送内容（打包） | 接收方补齐后 |
| :--- | :--- | :--- |
| 节点 0 → 节点 1 | Sum0, Sum1, Sum3 | 节点1 补齐位置3，**获得全部 4 块** ✅ |
| 节点 1 → 节点 2 | Sum0, Sum1, Sum2 | 节点2 补齐位置0，**获得全部 4 块** ✅ |
| 节点 2 → 节点 3 | Sum1, Sum2, Sum3 | 节点3 补齐位置1，**获得全部 4 块** ✅ |
| 节点 3 → 节点 0 | Sum0, Sum2, Sum3 | 节点0 补齐位置2，**获得全部 4 块** ✅ |

---

### 🎯 AllGather 最终状态

| 节点 | 位置 0 | 位置 1 | 位置 2 | 位置 3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | ✅ Sum0 | ✅ Sum1 | ✅ Sum2 | ✅ Sum3 |
| **节点 1** | ✅ Sum0 | ✅ Sum1 | ✅ Sum2 | ✅ Sum3 |
| **节点 2** | ✅ Sum0 | ✅ Sum1 | ✅ Sum2 | ✅ Sum3 |
| **节点 3** | ✅ Sum0 | ✅ Sum1 | ✅ Sum2 | ✅ Sum3 |

**所有节点都拥有完整的汇总结果！** 🎉

## 3. AllReduce = Reduce-Scatter + AllGather

| 阶段 | 操作 | 数据变化 | 通信方向 |
| :--- | :--- | :--- | :--- |
| **Reduce-Scatter** | 边传边**累加**，边传边**丢弃** | 每节点从 N 块 → 1 块有效数据 | 数据逐渐“浓缩” |
| **AllGather** | 边传边**拼接**，边传边**积累** | 每节点从 1 块 → N 块有效数据 | 数据逐渐“扩散” |

**两个阶段合起来，就是完整的 Ring-AllReduce：**

> **Reduce-Scatter（浓缩）→ AllGather（扩散）**

前者把分散的数据“压”成各持一块，后者把各持一块“扩”成每人都有一份完整的。

## 4. All-to-All（全交换）

### 目标
每个节点向**所有其他节点**发送**不同的数据块**，同时从所有其他节点接收不同的数据块。本质是对数据矩阵做**转置**。

### 初始状态
每个节点持有 4 块数据，第 i 块是要发给节点 i 的数据：

| 节点 | 发给节点0 | 发给节点1 | 发给节点2 | 发给节点3 |
| :--- | :--- | :--- | :--- | :--- |
| **节点 0** | A0 | A1 | A2 | A3 |
| **节点 1** | B0 | B1 | B2 | B3 |
| **节点 2** | C0 | C1 | C2 | C3 |
| **节点 3** | D0 | D1 | D2 | D3 |

### 目标状态
每个节点收到来自所有节点的第 rank 块数据：

| 节点 | 收到的内容 |
| :--- | :--- |
| **节点 0** | A0（来自0）+ B0（来自1）+ C0（来自2）+ D0（来自3） |
| **节点 1** | A1（来自0）+ B1（来自1）+ C1（来自2）+ D1（来自3） |
| **节点 2** | A2（来自0）+ B2（来自1）+ C2（来自2）+ D2（来自3） |
| **节点 3** | A3（来自0）+ B3（来自1）+ C3（来自2）+ D3（来自3） |

---

### 实现方式一：成对交换（Pairwise Exchange）

节点数为 N 时，需要 **N-1 轮**成对交换。每轮中，节点 i 与节点 j 交换各自需要发给对方的数据块。

**第 1 轮：相邻节点交换（距离 1）**
- 节点 0 ↔ 节点 1：0 把 A1 给 1，1 把 B0 给 0
- 节点 2 ↔ 节点 3：2 把 C3 给 3，3 把 D2 给 2

**第 2 轮：距离 2 的节点交换**
- 节点 0 ↔ 节点 2：0 把 A2 给 2，2 把 C0 给 0
- 节点 1 ↔ 节点 3：1 把 B3 给 3，3 把 D1 给 1

**第 3 轮：距离 1 的节点再次交换（剩余数据）**
- 节点 0 ↔ 节点 1：0 把 A3 给 1，1 把 B2 给 0
- 节点 2 ↔ 节点 3：2 把 C1 给 3，3 把 D0 给 2

**3 轮后，所有数据到达目标节点。**

---

### 实现方式二：直接点对点（Direct Send/Recv）

当**单块数据非常大**时，NCCL 会优先选这个方案：**所有节点同时两两通信，一轮完成**。

以 4 个节点为例，节点 0 的行为是：

```text
节点 0 ──发送 A1 ──→ 节点 1
节点 0 ──发送 A2 ──→ 节点 2
节点 0 ──发送 A3 ──→ 节点 3
节点 0 ←─接收 B0 ─── 节点 1
节点 0 ←─接收 C0 ─── 节点 2
节点 0 ←─接收 D0 ─── 节点 3
